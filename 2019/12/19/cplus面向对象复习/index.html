面向对象程序设计概述

程序设计范型是指设计程序的规范、模型和风格。
面向过程： 程序 = 过程 + 调用
面向对象： 程序 = 对象 + 消息

面向对象程序设计的基本概念

对象由属性和行为组成
类是一组具有相同属性和行为的对象的抽象
类和对象之间的关系是抽象和具体的关系

面向对象程序设计的基本特征

  1. 抽象
  2. 封装
  3. 继承
  4. 多态（不同的对象收到相同的消息执行时执行不同的操作）

c++概述

函数原型

如果函数调用的位置在函数定义之前，则要求在函数调用之前必须对所调用的函数作函数原型声明。
主要目的是让C++编译程序进行检查。
语法：返回值类型 函数名 (参数表);
例如
int fun(int a,long b);
long fun(int,long);

内联函数

在函数返回值类型前加 inline，该函数就被声明为内联函数，又称为内置函数。
每当程序中出现对该函数的调用时，C++编译器使用函数体中的代码插入到调用该函数的语句处、同时用实参取代形参。
利用空间换时间，减少对函数的调用，提高运行速度。

函数的重载

条件：函数名相同，参数个数或者参数类型不同。
若只有返回值类型不同、不允许重载

作用域运算符

通常情况下、局部变量会在其作用域中具有较高的优先级、它将屏蔽全局变量。

强制类型转换

运算符new和delete

new 后返回的是地址或者是NULL

引用

引用是为变量起另一个名字
类型 &引用名 = 已定义的变量名;

指针是通过地址间接访问某个变量，引用是通过别名直接访问某个变量
不能创建数组的引用

通过使用引用返回函数值可以将该函数调用放在赋值运算符的左边

类和对象

类的概念和结构

类是一种用户自定义的抽象数据类型。
类代表了一批对象的共性和特征。类是对象的抽象，而对象是类的实例。

类中有两种成员，数据成员和成员函数

访问权限修饰符

  • public
  • private
  • protected 可以被本类和派生类的成员函数访问

数据成员

默认是private
数据成员可以是任何数据类，但是不能用自动(auto)、寄存器(register)或者外部(extern)进行说明

成员函数

两种方式

一、定义成普通成员函数

采用”在类声明中只给出成员函数的原型，而将成员函数的定义放在类的外部“的定义方式，是C++程序设计的良好习惯。
这种习惯方式不仅可以减少类体的长度，使类的声明简洁明了，还有助于把类的接口和类的实现细节分离。

二、定义为内联成员函数

构造函数

构造函数是一种特殊的成员函数、它主要用于为对象分配空间、进行初始化
构造函数与类同名，没有返回值，它不需要也不能像其他的成员函数那样显式调用，他是在定义对象的同时被自动调用的，而且只执行一次
若没有自己写构造，编译系统默认生成空参构造，若写了有参构造，空参构造不会自动生成

析构函数

不能重载析构函数，在用delete处理掉指针的时候，对象消除，也会调用析构函数
用来释放内存空间

对象访问

this指针

对象由数据成员和成员函数组成
对于每个对象的内存空间，存的只有数据成员
同一个类的所有对象的成员的成员函数代码合用一个
通过this指针，使成员函数能够辨别出当前调用自己的使哪个对象

c++为成员函数提供了一个名为this的指针，这个指针称为自引用指针。
每当调用一个成员函数，系统就把this指针初始化指向该对象。
简单来讲，this指向当前调用对象的地址

对象的赋值和复制

对象赋值是通过默认赋值运算符实现的

当类中存在指针时，可能会产生错误

拷贝构造函数是构造函数参数是本类对象的特殊构造函数
编译器会自动生成
完成对象的初始化，调用方式有两种

调用拷贝构造函数的三种情况

  1. 用一个对象初始化另一个对象
  2. 函数调用传参
  3. 函数返回值是类的对象

静态成员

为了实现一个类多个对象的数据共享

一般情况下、静态成员函数用来访问静态数据成员
没有this指针

友元函数

友元类

A被说明为B的友元类，类A的所有成员函数都成为类B的友元函数。

常类型

常数据成员可以是常量或者常引用，只能通过构造函数的初始化列表对数据成员进行初始化（用冒号和括号那个），其他任何函数都不能对该成员赋值

常成员函数是常对象对外的唯一接口，常成员函数不能修改对象的数据成员，也不能调用普通成员函数

派生类和继承

派生类对基类的访问规则

private < protected < public

私有继承下，基类的private没有继承到派生类，其他都有继承，不过访问全变为private

派生类的构造函数

关于构造函数的执行顺序:

  1. 首先执行基类的构造函数(处于同一层次的各个基类的构造函数的执行顺序，取决于声明派生类时所指定的各个基类的顺序，严格按照声明时的从左到右)
  2. 执行内嵌对象的构造函数(取决于声明的顺序)
  3. 派生类的构造函数

多态性

相关概念

面向对象方法中，所谓多态性就是不同对象收到相同消息时，产生不同的行为

c++中，多态性的实现和联编(也叫绑定)这一概念有关。源程序经过编译、连接，成为可执行文件的过程就是把源代码联编在一起的过程。

运行前就能完成的联编称为静态联编。
静态联编支持的多态性称为编译时多态性，通过函数重载和模板实现

在运行时才能完成的联编叫做动态联编。
动态联编支持的多态性称为运行时多态性，通过虚函数来实现。
